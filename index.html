SseEmitter 적용 실시간 정보 스트리밍 구축

증상: 멀티 스레드 이벤트 전송 시 `ConcurrentModificationException` 무작위 발생으로 서버 다운타임 직면.

원인: 표준 `ArrayList`로 클라이언트 구독 리스트를 순회하던 중 타 유저의 네트워크 차단으로 요소를 수정/삭제하려 해 충돌 발생.

해결책: Thread-safe 구조인 `CopyOnWriteArrayList`로 전환하여 동시 순회 안전성 완전 보장 조치.

// Before
private final List<SseEmitter> emitters = new ArrayList<>();

// After
private final List<SseEmitter> emitters = new CopyOnWriteArrayList<>();
